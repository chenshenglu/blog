关于互联网金融和支付

发表于 2015-03-12

用户真正的金融需求

最近在微信上看到一篇文章，名字叫”九张图看懂互联网金融模式，召唤神龙！”名字很霸气，就像要集齐全部龙珠一样，文章比较简单，但是还是有点内容，至少告诉了你如何去思考。
我一直都比较喜欢去看那些基础的东西，因为所有的产品，不管多么复杂，其实都是从最基本的推导出来。只有掌握了最基本的，整个产品也了解得差不多了。

从用户需求的角度出发，互联网金融和传统金融其实没什么差，一样是为了满足用户的三大基本金融需求：投资、融资、支付。

其中提到了互联网的关键思维，就是从用户的需求去思考。金融产品不管包装得怎么炫丽，但是还是基于这些最基本的需求。用户的三大基本金融需求中，其实很简单。投资就是想”钱生钱”，资金富余者通过投资赚更多的钱。融资就是想”钱买钱”，资金短缺者通过支付利息获得钱。支付就是”钱换钱”，不同价值之间的交换。在这里，钱不应该理解为基本的货币，应该认为是价值，货币也是价值的体现。

基本关系

从用户的三大基本需求，我们有以下关系：
金融投融资：资金富余者==>资金短缺者
金融支付：用户<==>用户

推导衍生

可以看出，支付才是每个人都需要的，更加普遍存在,投融资只有在有资金的需求的情况下才会发生。考虑到生物进化的原则，事物的发展都是往高级发展的。因此，支付和投融资也在不断的进化，但本质还是不变。
传统的投融资需要银行，因为信息的不对称，导致资金短缺者和资金富余者双方存在不一致。银行的作用就是提供担保，向双方提供服务，保证投融资的资金流能够发生，并收取一定的费用。这就是传统的金融。
互联网金融：互联网金融的目的就是弱化中介的作用，更多以平台的形式，有时候甚至能够实现用户和用户之间类似支付一样的金融活动。原来需要银行等金融机构搭建的平台现在由互联网公司来维护，从另一个方面来说，资金短缺者融资成本降低，资金富余者利润增加，但双方都增加的了风险。收益从来都伴随着风险，高收益必然需要去面对高风险。
金融互联网：金融互联网就是传统的金融服务从线下搬到线上来，本质上没有变化。

HEXO基本操作

更换主题

  1. 找到喜欢的主题
  2. 复制git地址
  3. 执行代码
  4. 更改yml配置的主题
    git clone gitaddr theme/themename

发布到GITHUB

  1. 下载安装gitbash
  2. 生成sshkey并绑定到github账户
  3. hexo g & hexo d
  4. 上传成功后在setting页面获得page的地址，直接访问发现css的格式不对，url要设置为发布的地址，root设置为文件目录

    修改 _config.yml 中的 url 和 root 設定：
    url: http://chenshenglu.github.io/blog
    root: /blog/

  5. 访问正常

MarkDown基本语法

发表于 2015-03-04

MarkDown语法简洁，下面只整理一些常用的：

1.标题

语法如下：

#一级标题
#二级标题
######六级标题

效果如下：

一级标题

二级标题

六级标题

2.列表

语法如下：

* 选项1
* 选项2
* 选项3

1. 选项1
2. 选项2
3. 选项3

效果如下：

无序列表

  • 选项1
  • 选项2
  • 选项3

    有序列表

  1. 选项1
  2. 选项2
  3. 选项3

3.引用

只需要在文本前加入>这种尖括号（大于号）即可
语法如下：

> 这是一个引用

效果如下：

这是一个引用

4.图片和链接

语法如下：

图像:![百度logo](https://www.baidu.com/img/bdlogo.png)
链接:[HEXO](http://hexo.io/)

5.粗体和斜体

Markdown 的粗体和斜体也非常简单，用两个包含一段文本就是粗体的语法，用一个包含一段文本就是斜体的语法。
语法如下：

**这是粗体**
*这是斜体*

效果如下：
这是粗体
这是斜体

6.表格

表格直接采用html的语法









NAME ID MEMO
aaron
082512
“it is my name”

7.代码框

语法如下：

 printf(“hello world”);

printf("hello world");

8.分割线

3个”*”号

9.其它

反斜杠用于输出MD本身的代码
***
\
